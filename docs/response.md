2. 基于灵巧操作算法生成的姿态有多个候选，需要添加一些约束才能保证指尖传感器接触（从抓取操作来讲是否合理？）；
答：基于灵巧操作算法生成的姿态有多个候选，其中可能存在指尖传感器接触或指尖传感器不接触的抓取姿态。如果生成了指尖传感器不接触的抓取姿态，由于无法利用指尖传感器的信息，则直接使用灵巧手的位置模式进行抓取。若想利用指尖传感器的信息进行自适应抓取，则需要添加一些约束，例如指尖传感器的法向量与接触面的法向量反向等。

3. 分布力是否有用？小面积接触合力小，单点力大怎么办？
答：触觉传感器中存在分布阵列点，每个阵列点都能输出力的xyz三个分量。合力只知道“用了多大力”，阵列分布力知道“力在手指上的分布”。在抓软、小、易滑、易变形物体，分布力可以知道手指的哪个位置受力、受力是否均匀、是否偏离手指心、是否滑移、是否捏变形。

    分布力在抓软小物品的优势：
    1）合力看不出“偏心/抓歪”，分布力可以
    2）合力通过切向力的方差，事后判断；分布力可以通过检测压力中心持续移动，事前判断滑移。
    3）合力看不出接触面积，分布力可以知道多少阵列点受力，然后计算接触面积，从而计算压强
    4）合力是把所有接触点的力“打包”成一个向量，但丢失了向量起始点的信息

    目前抓取的方案先通过合力中切向力的方差进行滑动判断。

4. 如果算法识别的话（视觉+触觉），那么原始数据的信息应该不是一个中间值stiffness，那么这个转换是否还以必要（神经网络直接端到端？）。
答：原始数据的的stiffness是一个中间值，用来计算法向力阈值（防止法向力过大使物品损坏）和滑动阈值（切向力方差大于该值时，判断物品滑动）。在后续算法识别中，可以直接舍弃中间值stiffness，直接输出相关值。


5. 为什么闭合找接触阶段不用电机的位置模式或电机的速度模式
答：为什么位置模式/速度模式不适合闭合找接触阶段？
（1）位置模式：其核心原因是不知道目标的角度。不同物体、形状差异很大，无法预设一个统一的闭合姿态。闭合找接触阶段目标角度设小了，夹不住，设大了，手指会挤压物品。通过触觉传感器的方式停下，其依然存在电机刹车时间，电机的惯性会导致过冲，对于易损物品，存在损坏风险。
（2）速度模式：下发了一个速度指令后，电机按给定速度旋转，检测到接触后，需要下发速度=0的命令。同理，存在电机刹车时间，电机的惯性会导致过冲，对于易损物品，也存在损坏风险。
为什么力矩模式适合闭合找接触阶段？
（3）力矩模式，其指令本质就是电流的上限，即使没有触觉反馈，电机输出力矩也不会超过设定值。手指碰到电机时，外部阻力使电机自然减速停转。

6. 若位置/速度模式可以通过限定电流的方式，来进行找接触，为什么不采用？
答：限制电流≠控制电流，只要没有达到参考位置/参考速度，位置/速度模式是顶着限定电流的上限来跑，减速到零依然存在过冲。而力矩模式是随着外部阻力的增大，电机的控制电流减小，慢慢减速的过程。如果限幅电流设置很小，过冲也会随之减小。

    值得指出的是我们的灵巧手位置模式的同时，既可以下发限幅速度，也可以下发限幅电流，当外部阻力等于限幅电流产生的动力时，电机也会停下。通过设置电流的限幅，也能实现类似电机力矩模式的效果。
